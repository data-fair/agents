# Anonymous-action token on anonymous gateway/summary — Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Require anonymous callers of `/api/gateway/.../v1/chat/completions` and `/api/summary/...` to present a simple-directory anonymous-action token, verified via SD's JWKS, gated by a config flag (default on).

**Architecture:** A new pure-ish API service `assertAnonymousActionToken(req)` reads the `x-anonymous-token` header and verifies it with the already-initialized `session.verifyToken` (SD JWKS). Both routers call it inside their existing `if (!authenticated)` branch, after the role-quota check. The browser client fetches the token from SD only when anonymous and sends it as a header. A config flag `requireAnonymousActionToken` (default `true`) controls enforcement.

**Tech Stack:** TypeScript, Express, `@data-fair/lib-express` (session/JWKS), Vue 3 composables, `@ai-sdk/openai`, Playwright test runner, node-config + JSON-schema-generated config types.

## Background facts (verified, do not re-discover)

- SD endpoint (browser): `${$sdUrl}/api/auth/anonymous-action` where `$sdUrl` is exported from `ui/src/context.ts` (`= $sitePath + '/simple-directory'`). Returns the raw JWT as the response body.
- SD endpoint (tests): `directoryUrl + '/api/auth/anonymous-action'` (`directoryUrl` from `tests/support/axios.ts`).
- Token payload: `{ anonymousAction: true, ... }`, RS256, verifiable via `session.verifyToken(token)` (throws `NotBeforeError` / `TokenExpiredError` / signature errors).
- `session` and `httpError` are exported from `@data-fair/lib-express`. The session singleton is already initialized in `api/src/server.ts:23` (`session.init(config.privateDirectoryUrl)`).
- `config` is imported as `#config` (alias → `api/src/config.ts`), validated against `api/config/type/schema.json`. Booleans from env follow the existing `observer.active` / `OBSERVER_ACTIVE` pattern.
- UI anonymous detection: `useSession()` from `@data-fair/lib-vue/session.js` exposes `user` (computed); anonymous ⇔ `!session.user.value`.
- Only anonymous UI caller is `ui/src/composables/use-agent-chat.ts`. `_dev/summary.vue` is authenticated-only (`useSessionAuthenticated`) — no client change.
- Existing api tests never exercise the anonymous gateway branch, so enforcement does not break them. The summary test "should fail when not authenticated" uses default quotas (anonymous limit 0 → 403 from the role-quota check, which runs *before* the token check) and stays green.

- [ ] **Step 3: Create the verification service**

Create `api/src/anonymous-token/service.ts`:

```ts
/**
 * Verifies the simple-directory "anonymous action token" presented by anonymous
 * callers of cost-bearing endpoints (gateway, summary). The token is issued by
 * SD's rate-limited /api/auth/anonymous-action endpoint and verified here via
 * the JWKS the session singleton was initialized with in server.ts.
 */
import { type Request } from 'express'
import { session, httpError } from '@data-fair/lib-express'
import config from '#config'

export async function assertAnonymousActionToken (req: Request): Promise<void> {
  if (!config.requireAnonymousActionToken) return

  const token = req.get('x-anonymous-token')
  if (!token) throw httpError(401, 'anonymous action token required')

  let decoded: any
  try {
    decoded = await session.verifyToken(token)
  } catch (err: any) {
    if (err?.name === 'NotBeforeError') throw httpError(401, 'anonymous action token not yet valid')
    if (err?.name === 'TokenExpiredError') throw httpError(401, 'anonymous action token expired')
    throw httpError(401, 'invalid anonymous action token')
  }

  if (decoded?.anonymousAction !== true) throw httpError(401, 'invalid anonymous action token')
}
```

- [ ] **Step 1: Create the token composable**

Create `ui/src/composables/use-anonymous-token.ts`:

```ts
import { $sdUrl } from '~/context'

// Module-level cache: a single anonymous-action token is reused across the app.
let cached: string | null = null
let inFlight: Promise<string> | null = null

export async function getAnonymousToken (): Promise<string> {
  if (cached) return cached
  if (inFlight) return inFlight
  inFlight = (async () => {
    const res = await fetch(`${$sdUrl}/api/auth/anonymous-action`)
    if (!res.ok) throw new Error(`failed to get anonymous action token: ${res.status}`)
    const token = (await res.text()).trim()
    cached = token
    return token
  })()
  try {
    return await inFlight
  } finally {
    inFlight = null
  }
}

export function resetAnonymousToken (): void {
  cached = null
  inFlight = null
}
```
